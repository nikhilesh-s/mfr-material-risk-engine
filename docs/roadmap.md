# Roadmap

## Phase 2

Status: current release posture

Phase 2 is the engineering prototype stage.

Focus areas:
- freeze deterministic inference behavior,
- stabilize the model-serving interface,
- document system constraints,
- make interpretability outputs usable for technical review,
- support controlled demonstrations and early evaluator feedback.

## Phase 3

Target: pilot validation

Planned focus:
- broader pilot evaluation with representative users,
- stronger comparison against physical test outcomes,
- improved uncertainty characterization,
- tighter data-governance and traceability practices,
- clearer benchmark protocols for technical review.

## Phase 4

Target: workflow integration

Planned focus:
- integration into engineering material-selection workflows,
- better input-data management,
- structured experiment tracking,
- revision-controlled model and dataset governance,
- deployment hardening for team use.

## Phase 5

Target: multi-property screening expansion

Planned focus:
- extension beyond fire-risk screening into adjacent material-screening properties,
- multi-objective candidate comparison,
- richer what-if analysis across material and coating combinations,
- broader decision-support support for design trade studies.

## Roadmap Principle

Each phase should increase system utility without blurring the core boundary: Dravix remains a screening and prioritization tool unless and until a future validated program explicitly broadens its scope.
